# 1. Record architecture decisions

Date: 2026-07-04

## Status

Accepted

## Context

We need to record the architectural decisions made on this project so that future
contributors (and future us) understand *why* things are the way they are.

## Decision

We will use Architecture Decision Records, as described by Michael Nygard, stored as
numbered Markdown files in `docs/adr/`.

## Consequences

Each significant decision gets a short, immutable record. Superseded decisions are
marked as such rather than deleted.
